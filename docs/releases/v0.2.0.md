# Bader Charge Analyzer v0.2.0

v0.2.0 引入统一的“已应用分析范围”：完整科学数据始终保留，而数据表、统计、二维图、导出和三维电荷着色共享同一份已提交的原子选择。

## 分析范围

- 目标原子输入首先是草稿。编辑草稿不会立即筛选已有结果；点击“应用范围并分析”后才原子性地切换所有相关视图。
- 空范围安全地表示全部原子，不再强制要求先定义片段。
- 片段是可选的高级统计，并可引用当前范围之外的原子；它不会限制或阻止基础计算。
- 数据表、统计与二维对比使用同一已应用范围。多体系比较按原子编号对齐，避免按行位置错配。
- 导出分为“范围结果”和“完整原始结果”：前者对应当前已应用范围，后者始终包含全部原子数据。

## 三维结构与缓存

- 三维视图保留体系的完整结构和成键关系，只有目标原子参与 Bader 电荷转移颜色标注和颜色范围归一化。
- 几何结构与着色外观分离。应用新的原子范围时，通常只更新目标原子的颜色，不重新构建结构和化学键。
- 场景缓存按工作区、结构修订、分析修订和渲染设置进行版本安全隔离。因此，在数据表、二维图和三维视图之间往返不会反复渲染同一场景，不同体系也不会错误共用缓存。
- 更换体系会读取该工作区自己的缓存；结构源文件改变或相应修订变化时，旧几何缓存失效。分析范围或电荷结果改变时，外观版本更新。缓存达到容量上限时会释放较旧场景。

## 旧项目迁移与兼容

- 旧版 `results.json` 中的全原子结果继续完整保留，不会因升级而裁剪或覆盖。
- 工作区状态缺少 v0.2.0 新增元数据时，会安全默认使用全部原子，并为结构修订和分析修订初始化兼容值；首次保存后写入新元数据。
- 旧项目中已有的片段和结果仍可继续使用。片段现在作为可选统计存在，不会成为重新分析的前置条件。
- 迁移不改变原始 `ACF.dat`、`CONTCAR` 或 `POTCAR` 文件。

## 使用提示

输入目标范围后，先确认界面提示存在未应用更改，再点击“应用范围并分析”。若要恢复全部原子，可清空范围后重新应用。需要完整归档时请选择“完整原始结果”，而不是范围结果。
