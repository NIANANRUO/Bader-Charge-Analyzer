# Bader 分析范围统一与 3D 缓存设计

## 1. 背景与问题

当前程序虽然允许输入目标原子和定义片段，但“完整计算结果”“当前分析范围”和“各视图展示范围”没有明确分层：

- 后台生成全部原子的 Bader 结果，目标表达式只在部分界面路径生效。
- 单体系表格可能即时过滤，而多体统计、部分二维图、导出和 3D 仍使用全部原子。
- 3D 面板没有接收已提交的目标范围，因此所有原子都参与 Bader 电荷着色。
- 片段功能位于主分析流程中，给用户造成“不定义片段就不能计算”的强制感。
- 每次进入 3D 标签页都会发起强制同步；结构几何和电荷表现未分层，导致不必要的等待。
- 现有 3D 数据缓存依赖 Python 对象地址，无法稳定表达结构或分析数据是否真正变化。

本设计统一上述语义，同时保留完整科学数据、完整 3D 结构上下文和多体系批量分析能力。

## 2. 设计目标

1. 将目标原子定义为唯一的“当前分析范围”。
2. 点击“计算 / 分析”后，表格、统计、二维图、摘要、普通导出和 3D 电荷表现同时切换到同一分析版本。
3. 输入目标表达式时只修改草稿，不即时改变任何已有结果。
4. 目标留空表示全部原子；片段完全可选，不影响主分析是否可执行。
5. 保留全部原子的原始 Bader 结果，允许随时切回全部原子或导出完整结果。
6. 3D 保留完整结构，但只有当前目标原子参与 Bader 电荷着色和色标归一化。
7. 表格、二维图和 3D 标签页往返切换不触发结构重建或重新着色。
8. 每个体系拥有独立、可验证的 3D 缓存；更换体系时不错误复用其他体系场景。
9. 多体系批量分析支持公共默认目标范围以及逐体系覆盖。

## 3. 非目标

- 不改变 Bader 电荷的物理定义和现有净电荷计算公式。
- 不丢弃或永久裁剪完整原子结果。
- 不让片段替代目标范围，也不让目标范围限制片段定义。
- 不在本次设计中扩展新的原子选择语言；现有编号、范围和元素选择语义继续使用。
- 不以全局“最近一次 3D 画面”作为跨体系缓存。

## 4. 核心概念

### 4.1 完整结果

`full_result` 是某个工作区全部原子的解析和电荷计算结果。它是所有分析投影的事实来源，应保存到磁盘并在会话中缓存，不受目标范围裁剪。

### 4.2 分析范围

`committed_scope` 是最近一次成功点击“计算 / 分析”后生效的目标表达式。解析后的 `selected_atom_ids` 是所有常规结果视图共同使用的原子集合。

空表达式解析为该体系的全部有效原子。

### 4.3 草稿范围

`draft_scope` 是输入框中尚未提交的文本。修改它只产生“有未应用更改”的界面状态，不改变任何已提交结果。

### 4.4 片段

片段是用户命名的可复用原子集合，用于额外统计。片段直接基于 `full_result` 计算，可以包含当前分析范围之外的原子，并且不是主分析的必填项。

## 5. 状态模型

每个工作区维护独立的 `AnalysisSession`：

```text
AnalysisSession
├─ workspace_id          工作区唯一标识
├─ source_revision       输入文件集合的稳定版本
├─ full_result           全部原子的完整结果
├─ structure             完整结构
├─ draft_scope           尚未提交的目标表达式
├─ committed_scope       最近成功提交的目标表达式
├─ selected_atom_ids     已解析且排序的目标原子编号
└─ analysis_revision     每次成功提交后递增的分析版本
```

`source_revision` 至少覆盖结构文件、`ACF.dat` 和 `POTCAR` 的有效版本。实现采用规范化绝对路径、文件大小和纳秒修改时间组成快速指纹；检测到路径或元数据变化时重新计算对应文件的 SHA-256 内容指纹并使旧结果失效。Python 对象地址不得作为版本语义。

`analysis_revision` 只在一次分析范围成功提交后递增。输入草稿、切换标签页和打开片段管理器都不能改变它。

## 6. 组件边界

### 6.1 `SelectionResolver`

唯一负责解析目标表达式，输入为表达式和体系完整原子信息，输出为排序后的原子编号集合。它统一处理空表达式、范围倒序、越界、未知元素和未匹配任何原子等情况。

所有表格、绘图、统计和 3D 模块禁止自行重复解析目标字符串。

### 6.2 `AnalysisSession`

保存单个体系的完整数据、草稿状态、已提交范围和版本。它不负责 Qt 控件更新或 3D 渲染。

### 6.3 `AnalysisSessionStore`

按工作区管理会话，提供批量验证、整体提交和整体回滚。多体系提交必须先验证所有目标表达式；只有全部通过后才一次性替换所有相关会话的已提交状态。

### 6.4 `AnalysisProjection`

从 `full_result` 和 `selected_atom_ids` 派生目标范围 DataFrame、摘要统计和元素聚合。表格、二维图和普通导出均使用同一个投影接口。

### 6.5 `SceneCache`

按工作区管理 3D 结构层和表现层版本。它只负责可视化资源的复用与释放，不参与科学计算，也不决定目标范围。

## 7. 提交流程

1. 用户编辑公共目标表达式，或在批量设置对话框中添加逐体系覆盖。
2. 界面只更新 `draft_scope` 并提示“有未应用更改”。
3. 用户点击“计算 / 分析”。
4. `AnalysisSessionStore` 为每个选中体系确定最终表达式：逐体系覆盖优先，否则使用公共默认值。
5. 使用 `SelectionResolver` 验证全部表达式并生成目标编号集合。
6. 如任一体系验证失败，整批提交终止，所有视图和已提交会话保持不变，并显示具体体系、表达式和错误原因。
7. 全部验证成功后，执行必要的文件解析和 Bader 数据计算。已经存在且 `source_revision` 未变化的完整结果可以复用。
8. 所有体系均成功后，原子化提交新的 `committed_scope`、`selected_atom_ids` 和 `analysis_revision`。
9. 由同一个已提交版本刷新表格、统计、二维图、摘要、普通导出上下文和 3D 表现层。
10. 任一体系的实际分析失败时，本次批量操作不提交任何体系的新范围或新 `analysis_revision`，所有可见视图继续使用整批操作前的成功会话。过程中已经生成的完整原始结果可以作为内部缓存保留，但在下一次整体提交成功前不得进入用户可见投影。

## 8. 界面设计

右侧主分析区调整为两个职责明确的区域：

```text
分析范围
目标原子：[72-74                  ]
当前生效：72-74（3 个原子）
[使用全部原子]  [批量设置…]

高级分析（可选）
[管理片段…]
已定义 1 个片段
```

交互规则：

- 输入目标表达式后显示“有未应用更改”，但现有视图保持不变。
- “使用全部原子”将草稿设为空；仍需点击分析按钮后生效。
- 批量设置对话框采用“公共默认值 + 单体系覆盖”，并显示每个体系解析后的原子数。
- 当前生效范围持续显示在主面板或状态栏，不能仅依赖输入框内容判断。
- 片段入口标记为“高级分析（可选）”，无片段时计算按钮仍正常可用。
- 尚无结果时按钮显示“计算 / 分析”；草稿变化后显示“应用范围并分析”；无变化时显示“重新分析”。

## 9. 各视图一致性

### 9.1 单体系表格

只显示 `AnalysisProjection` 中的目标原子。恢复全部原子范围后显示完整结果。

### 9.2 多体系统计表

每个体系使用自己的 `selected_atom_ids`。跨体系数据必须按真实原子编号对齐，禁止按 DataFrame 行位置对齐。某体系没有对应目标原子时显示缺失值符号，不得把其他原子的值错配到该行。

摘要行只使用各体系自身的目标原子数据。

### 9.3 摘要与元素汇总

原子数、总和、均值、标准差、最大增益、最大损失和元素聚合均基于目标投影计算。

### 9.4 二维图

原子级图表只接收目标投影。元素级图表先应用目标范围，再按元素求和或均值。目标表达式不由绘图模块再次解析。

片段级图表仍使用片段定义和完整结果，不受当前目标范围裁剪。

### 9.5 导出

普通 CSV、当前视图图片和当前统计导出遵守已提交目标范围。另提供明确命名的“导出完整原始结果”，用于获取全部原子数据。

### 9.6 片段统计

片段统计直接基于 `full_result`，可以引用当前目标范围之外的原子。片段定义或统计错误只影响对应片段，不阻止主分析。

## 10. 3D 渲染设计

### 10.1 分层

3D 场景分为两个生命周期：

```text
结构层（昂贵、稳定）
原子几何 + 化学键 + 晶胞 + 相机状态

表现层（轻量、经常变化）
目标原子集合 + 电荷值 + 色图 + 透明度 + 标签
```

完整结构始终保留作为空间上下文。当前目标原子使用 Bader 电荷色图；非目标原子沿用现有完整结构显示方案，但不参与 Bader 色标归一化。色条极值、分位数和 Gamma 映射只基于目标原子电荷。

### 10.2 更新规则

- 首次打开某个体系的 3D 视图时构建结构层。
- 表格、二维图和 3D 标签页之间切换时直接显示现有场景，不发起数据同步、结构重建或重新着色。
- 修改目标范围并重新分析时，如果结构未变，只更新目标集合、电荷值、颜色和色标。
- 修改色图、Gamma、透明度、标签等表现设置时，只更新对应视觉属性。
- 结构文件、原子数、元素序列、工作区集合或几何相关设置变化时，才重建受影响体系的结构层和化学键。
- 表现层更新必须保留相机角度、缩放、选中原子和最大化体系状态。

### 10.3 缓存键

```text
GeometryKey =
  workspace_id
  + structure_fingerprint
  + atom_count
  + element_sequence

AppearanceKey =
  analysis_revision
  + selected_atom_ids
  + charge_revision
  + render_settings
```

`GeometryKey` 相同时复用结构层；只有 `AppearanceKey` 变化时轻量更新表现层。`GeometryKey` 变化时只废弃并重建对应体系缓存。

每个体系拥有独立缓存。切换到另一体系时必须先验证该体系自己的缓存键，不能复用全局最近场景。默认最多保留 6 个体系的结构缓存；超过上限时按最近最少使用顺序释放未显示体系。当前正在显示的体系不得被淘汰。删除工作区、替换结构文件或退出程序时立即释放对应 VTK 资源。

## 11. 错误处理与恢复

- 目标表达式非法、越界或未匹配原子：拒绝整批提交并保留旧结果。
- 某体系缺少必要输入或分析失败：不覆盖该体系上一次成功会话。
- 3D 缓存校验失败：仅重建对应体系，不影响表格和二维结果。
- 3D 渲染失败：显示可重试提示；科学数据和其他视图继续可用。
- 片段定义错误：显示片段级错误，不改变主分析状态。
- 从磁盘恢复项目时，如果输入文件版本与已保存会话不匹配，应将完整结果标记为过期，要求重新分析，而不是静默复用。

## 12. 性能约束

- 标签页切换造成的结构重建次数必须为零。
- 仅目标范围变化时，化学键检测次数必须为零。
- 同一体系、同一结构版本再次进入 3D 时必须命中结构缓存。
- 切换到未缓存体系时可以首次构建；后续返回且版本未变时必须复用。
- 单一体系缓存失效不得连带重建其他未变化体系。

## 13. 测试与验收标准

### 13.1 选择与会话测试

1. 目标留空解析为全部原子。
2. `72-74` 解析为三个稳定排序的编号。
3. 输入草稿不会改变已提交范围或任何视图投影。
4. 成功提交后所有投影使用同一 `analysis_revision`。
5. 任一体系表达式非法时，批量提交整体回滚。
6. 公共默认范围和逐体系覆盖均正确解析和保存。

### 13.2 视图一致性测试

1. 提交 `72-74` 后，单体系表格只显示三个原子。
2. 摘要、元素汇总和二维图只使用这三个原子。
3. 多体系表按原子编号对齐，不按行位置错配。
4. 普通导出只包含目标原子，完整原始结果导出仍包含全部原子。
5. 未定义片段时可正常分析。
6. 片段可以统计目标范围之外的原子。

### 13.3 3D 测试

1. 3D 保留完整结构，只有目标原子参与 Bader 着色和色标范围。
2. 表格、二维图和 3D 往返切换不调用结构重建。
3. 目标从 `72-74` 改为 `65-68` 时只更新表现层。
4. 切换体系不会复用其他体系的结构缓存。
5. 替换 `CONTCAR` 或 `POSCAR` 后对应 `GeometryKey` 变化并触发重建。
6. 只替换 `ACF.dat` 或目标范围时结构层不重建，但表现层更新。
7. 表现层更新后相机和选择状态保持不变。
8. 删除工作区或退出程序时释放对应 VTK 资源。

## 14. 实施边界

该设计可在一个实施计划内完成，但应分阶段推进：先引入会话与投影模型并统一二维视图，再调整片段入口，最后分离 3D 结构层和表现层并加入稳定缓存键。每一阶段都必须维持完整结果可恢复，并以自动化测试证明范围一致性和缓存正确性。
